Some html-things

Here are some basics you might want to know about html:

  • Text between <h1> and </h1> is shown as a big title. This 'h1'-tag is only used once a page mostly
  • Text between <h2> and </h2> is shown as a less big title than what the h1-tag offers and is used more often
  • Text between <h3> and </h3> is shown as a even lesser big title.
  • The tag <br> creates a break (A new line)
  • The text between <em> and </em> is italic
  • The text between <strong> and </strong> is shown bigger
  • An unordered list can be created by using <ul> and </ul>
    To list things, between these tags, create <li> and </li> and write the list-item between it
  • The tag <abbr> or </abbr> is creating showing the website, that this is an abbriviation. It requires the attribute title followed by the explanation of the abbriviation, which is visible in the website by hovering above the abbriviation. For example:
    <abbr title="by the way">btw</em> gives you following output:
    btw
  • Text between <address> and </address> is counted as an address by the browser. For example:
    60 Broadway, London, W81 0QI
  • <section id=""> and </section> with everything the section should include in between these tags creates sections you can refer to with the <a href="id"> and </a> tags by changing id with # followed by the id of the section is
  • <a href="link"> and </a> creates a clickable thing that leads you to whatever link is.
  • <a href="link" download> and </a> creates a clickable thing that lets you download whatever link is:
    Download Screenshot
  • <a href="mailto:link"> and </a> creates a clickable thing that lets you email whatever link is as long as mailto: is mentioned:
    Email me at email
  • <a href="tel:link"> and </a> creates a clickable thing that lets you call whatever link is as long as tel: is present:
    Call Me
  • <a href="link" target="_blank"> and </a> creates a clickable thing that lets you open whatever link is in a new tab:
    Open Google
  • With &, you can create some special things for your application, based on the string-"code" that follows. For example:
     nbsp; creates a non-breaking-space, so a space
